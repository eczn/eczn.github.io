# 注意到... ↵

当程序执行读取某个负数的时候，其实是加载了一个对应相反数的二进制补码，因此 -2 & 0xff 结果是 0b11111110 为 254
因此如果给你一段内存，从字节流离很难判断说这个到底是不是正数，因为有符号数是被定义出来的，从外表上看跟一个无符号数一模一样。
另外，建议位运算都用 BigInt 来完成，有比普通 number 更好的整数特性
