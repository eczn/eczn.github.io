的缩写，比如腾讯云，阿里云这些，他们提供这些基础设施来进行生产活动。 \n\n租/卖给你服务器，你直接可以使用，机器的硬件运维无需你关注，他们会做好。 \n\n## PaaS \n\nPaaS（Platform as a Service）更进一步，比如微软的VS，或者Coding提供的Git服务，又或者是七牛提供的数据存储服务等等。 \n\n总之，这部分你编程中需要用到，但是自己实现又麻烦的，而别人有偿帮你做了这些，即服务提供商把互联网资源转化成可编程的接口给你使用。 \n\n\n## SaaS \n\nSaaS（Software as a Service）软件即服务，这部基本算是成型了，把软件作为服务出售。 \n\n比如你需要一个 Web 存储服务，但是你不想要自己搭服务器，也不想要自己编程做，所以你可以用百度云提供的存储服务，直接使用。 \n\n或者石墨那样的提供的多人在线文档协作也算是 SaaS。 \n\n## BaaS \n\nBass 即 Backend as a Service，后端即服务。 \n\n就是给钱帮你把后端的事情做了，比如 leanCloud 这样的。 \n\n跟 PaaS 有点类似，BaaS 会把互联网资源转化成可编程的接口给你使用，不过这里要更具体一些： \n\n> 会把后端转化成可编程的接口给你使用 \n\n也就是他们造了个后端系统给你直接使用。 \n\n\n\n# Why ? \n\n有了这些服务，一个普通的开发者就可以用上顶级存储服务，可以配备专业的运维工程师帮助管理服务器，好用的轮子（短信推送这些）\n\n其实都是云带来的好处，我理解的云是一坨服务器的集合，对外提供服务，比如上述各种各样的 XaaS 。 \n\n可是为什么会出现这么多 XaaS ？ 其实都是市场供需的影响，毕竟完整的一套 Web 服务是极为复杂的： \n\n1. 服务器硬件管理 \n2. 数据库、Redis集群管理 \n3. 短信服务、邮件服务 \n4. 开发环境\n\n等等等等 .... \n\n就第一条来说，要做好就得请个懂Linux的、懂网络的、懂安全的来做 ... \n\n后面的一条接着一条也是如此，这样来看的话就连自己开个网站都很麻烦，因而出现了 IaaS
