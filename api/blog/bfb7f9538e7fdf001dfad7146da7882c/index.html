2]]\n```\n\n括号代表又一个计算式，它是递归的，而数组本身天然就是可构成递归结构的，因此可以无缝结合。 \n\n\n# 数组计算式 \n\n像上面提到的形如 ['+', 1, 2, 3] 可以叫做数组计算式\n\n那么要如何运行求值一个 `数组计算式` 呢？ 经过考虑，有如下三个点： \n\n1. 数组计算式的第一个元素是 `如何计算` \n2. 计算的过程应该是剩余元素 reduce 的过程 \n3. 数组计算式的元素也可以是数组计算式 \n\n考虑到第三点，需要用递归的方式去实现：\n\n``` js \nlet table = {\n '+': (...args) => args.reduce((acc, cur) => acc + cur)\n}\n// let fn = table['+']\n// fn(1, 2, 3); \n// => 1 + 2 + 3 = 6 \n\nlet calc = exp => {\n if (!Array.isArray(exp)) {\n // 不是数组，是数字\n // 则返回这个数字的计算结果即返回数字本身\n return exp;\n } else {\n // 说明是数组 \n\n // first 是 `+` \n // args 是剩余的参数 \n let [first, ...args] = exp;\n\n // 取出计算函数 \n let fn = table[first];\n\n // args 里的元素也可能是数组计算式 \n // 递归的调用自己 \n args = args.map(e => calc(e));\n \n // 最后进行 fn 的调用 \n return fn.apply(null, args);\n }\n}\n```\n\n![计算结果](/images/calc-result-array-exp.png) \n\n拓展 table，可以使其变得跟普通的计算式一样强大，我们甚至可以自定义运算符，仅需在 table 上编写函数即可。\n\n\n# 变量和计算式\n\n变量其实是一种映射，比如： \n\n``` js\n// 这是变量池，里面存着变量名和对应的值 \nlet vars = {\n 'a': -5 \n}\n\n// 在之前的 table 上拓展新的运算: 变量求值 \ntable.find = var_name => vars[var_name]; \n// let fn = table['find']\n// fn('a') \n// => -5 \n\n// 要计算： \n['+', 'a', 2, 3]; \n\n// 其实就是： \n['+', ['find', 'a'], 2, 3]; \n``` \n\n来看看执行结果： \n\n![带变量的计算结果](/images/calc-result-array-exp-var.png)\n\n结果为 `0`, 符合预期。 \n\n\n# 怎样写一个解释器 ?\n\n对于计算机的处理来说，数组计算式更好处理，对于我们人类来讲，普通的计算式更好理解。\n\n解释器的一个工作是把普通的计算式解释为数组计算式，并吧变量的搜索附加上去，形成一种数据结构，如果这种数据结构是 `汇编代码`，那么这个解释器叫做编译器比较妥当，机器可以直接运行汇编码；如果解释成某种中间结构，那么还是叫做解释器，不过这种中间结构是可以求值的（比如上面提到的数组计算式就是一种中间结构），求值它需要一台抽象的机器，这台机器需要我们编程实现，它叫做虚拟机。 \n\n\n解释器的编写，有两大问题： \n\n1. 如何把源码（比如计算式）变成某种中间结构（比如数组计算式）\n2. 构造虚拟机对中间结构进行求值 （这里的 calc 函数就是虚拟机）\n\n而当中最有趣的，是第二点，构造虚拟机运行这种中间结构。 \n\n--- \n\n\n对了对了，忘记说了，本篇的数组计算式这个名字是我瞎说的，它的学名应该叫做 S 表达式，是构造解释器的时候很重要的一种数据结构，它是一种可迭代的递归结构，刚好跟数组对应，因此可以用数组去表达一个 S 表达式。
