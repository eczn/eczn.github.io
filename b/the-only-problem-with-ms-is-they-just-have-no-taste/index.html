微软唯一的问题是没有品味
标题取自媒体对乔布斯的访谈，在访谈中，乔布斯认为：微软虽强，但微软唯一的问题是没有品味。
The only problem with Microsoft is they just
have no taste.
—— Steve Jobs
所谓 “品味”，我的理解 —— 产品重视用户体验，有高雅的 “产品审美”，有良知的 “产品动机”。
乔布斯在世纪交替的那个时代就说了这话，再来看看过去这二十年微软的产品，可见其预见力。本文着重对比 windows pc 和 apple mac 的几个小细节来理解为何乔布斯为何持有这种观点。

# 设置屏幕壁纸问题 ↵

多屏用户经常需要设置多个壁纸到多个屏幕上，这个时候就非常需要某种可以自由设置屏幕的功能了，这一点两家公司也是有自己不同策略：
>> 苹果设置多屏壁纸采取了弹出多个选项卡的形式来设置
>> 微软 win10 设置多屏壁纸需在设置的壁纸上右键点选才能触发设置
看起来似乎没有任何问题，但我作为用了十几年 windows 的我来说根本没有想到这地方居然还能按右键选择，核心的问题是一旦你不知道能右键你就会认为没有这个功能了，因此我网上搜的时候很多人认为微软不支持这个，但我想微软的程序员怎么可能不会有这个需求呢？

# 系统名词的选定 ↵

对于硬盘的操作，两家公司有不同的命名，分别是格式化和抹掉：
格式化 Format >> 专业词汇，意思是重建文件系统，强调过程。
抹掉 Erase >> 字面意思，抹掉硬盘数据，强调结果。
微软采用了格式化这个词，苹果采用了抹掉这个词。那么，对于大部分用户来说，哪个更好？
大部分用户（非专业人员）不需要关心格式化相关的计算机原理，他们需要的只是结果，因此用抹掉这个词会更好。(虽然我是程序员但我也认同使用抹掉会比格式化好)

# 系统设计僵化 — 以硬盘分区为例 ↵

windows 为什么需要硬盘分区，主要是以下三个考量：
>> A. 历史问题
过去的文件系统不支持容量大的硬盘，因此不得不搞分区，即一个硬盘上搞多个文件系统，这样就可以充分利用大容量硬盘了
>> B. 局部最优解
对于机械盘来说分区可以提升一定的性能。具体来说，盘片转速是固定的，因此盘片边缘速度慢点，而盘片内圈的速度快点，又由于磁盘寻址设计的缘故，内圈和外圈上可以放的数据总量是一样多的，这就造成了内圈的数据会密一点，外圈的会散一点，在内外圈存在线速度差异情况下，外圈能到的实际速度肯定是比内圈少的，而这里引入分区则可以实现内圈的高速空间多分给 C 盘，外圈的分给其他盘这样来提升 C 盘速度来提升系统响应速度 ~
>> C. 重装系统
系统重装后的时候怕自己的文件丢失，所以单独设置分区，这样格式化 C 盘的时候其他盘不受影响。
然而随着操作系统的发展，上述 ABC 三点已经有了成熟的方案了，第一点不用说了现在的文件系统都是以 PB 级的设计目标去设计的，作为用户我们只需要一个文件系统就可以了。
第二点在现在遍地 SSD 做系统盘的时代已经不重要了，SSD 是大趋势，价格也越来越低，操作系统有义务去推进这个趋势提高用户体验，而不是一昧迎合市场。
只剩下第三点问题，但是这个并不是问题，因为：为什么系统重装需要格式化硬盘，现在技术这么发达为什么不搞个不需要格式化就能重装的系统呢？，或者说即便要格式化也有办法保证不丢失用户数据~
这里提一下，苹果重装的时候只有在无法识别磁盘的时候才必须要抹掉，大部分情况可以不抹掉磁盘的。
反观分区会带来的问题：
  1. 用户理解困难，尤其电脑是出问题的时候，分区的概念我认为是区分本地磁盘和 U 盘光盘这些比较合理，本地硬盘一个就好，大部分用户分区来区分音乐、软件、视频等文件其实可以放在不同的文件夹来实现，而 win10 时代还是没想着淡化盘符这个设定，越走越远了。
  2. 应用安装后会强依赖盘符，一旦系统出现了盘符更改或盘符紊乱，程序就炸了。想象一下软件都在 D 盘，哪天重启后这个盘变成了 E 盘 … 如果你写过程序或者遇到过这个情况，就知道这个多蛋疼了 (我用 windows 十几年了, 这个盘符紊乱问题至少出现七八次了)
  3. 分区会导致系统盘不够用，导致系统无法正常使用。这个问题几乎是每一个 windows 用户一定会遇到的问题，尤其是在这个应用遍地开花的时代下更难受，其中特别地，很多傻逼应用还想尽办法的把它自己安装在你的 C 盘，即便你想办法确实将软件都装到了 D 盘，但软件运行产生的数据还是很喜欢放到你 C 盘，所以 C 盘会一直变少变少变少，直到没有 … 最后蓝屏卡死机

# 设计简陋的 win10 ↵

win10 丑是公认的事实，它无疑是 win 系列最丑的版本，各种色块。。。以及设计元素不统一，残留了大量 win7 win8 时代的设计元素，用过 win10 的都懂，值得一提的是 win10 都五六年了，还是没想着去掉旧系统的设计元素 … 微软肯定不是因为没钱来做好系统 UI，只是其认为视觉并不重要，品味可见一斑
设置界面混乱，有的人喜欢用新版的设置界面（左边的）也有的用右边的老版 “控制面板”，神奇的是，有时候左边新版并不满足需求，还得时不时的用右边的控制面板。
mac 一向以来设置界面没有大变化，一直以来都是这个风格，系统升级后绝大部分设计资源都会换成新的。

# 应用安装问题 ↵

Win 安装应用并不容易，需要理解很多概念之后才能正确安装应用，否则搞不明白安装过程中的下一步到底是什么意思 … 再然后电脑就不小心装了全家桶了
（题外话《我们中国人到底做错了什么要被安装全家桶？》）
而大部分 Mac 的应用安装非常有趣味，打开安装包后拖动到 Application 即可，这个我第一次用的时候没反应过来，原来安装应用如此简单 （或者可以走 App Store 安装）

# win10 和 “互联网思维” ↵

>> 强制更新
这个不解释了，懂得都懂，而且带起了业界强制更新的歪风邪气，我猜是互联网产品经理混进微软了。
反对的理由很简单，连手机都没敢上的强制更新你微软居然敢用在电脑这种强生产力工具上，打断用户工作进行强制更新并不能保证一定成功，更新失败了开不了机了就只能重装 …
另外，因为 win10 是全球几十亿人都在用的，搞强制更新一定会有人更新的时候卡的要死的情况，尤其是东亚这类偏远地区（dddd），一定有不少人更新失败不得不重装系统的。
类似的理由还有很多 … 微软也有自己的措施，比方选择凌晨更新，但我是熬夜党；比如选择用户电脑休眠的时候搞，但我程序员在跑后台编译 … 总之，你没法完全解决中断工作流带来的所有问题。
这么多人批评但微软还是做了，为何？因为可以做所谓的 “快速迭代”，但是生产力机器不是互联网应用，稳定性才是最重要的。
>> 强制登录
